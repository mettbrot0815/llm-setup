01  What it installs

Bare Ubuntu to full
AI workstation.

Every component wired up and aliased, ready on the first terminal open after install. Fully idempotent — safe to re-run after partial failures.

Step 01
Preflight
Checks sudo, OS, disk, internet. Installs all apt deps in one batch call to avoid dpkg lock races. Refreshes the model catalog in the background so step 06 sees the latest list.
Steps 02–03
Python + Node
Isolated venv for all Python packages with retry on dpkg race failures. Node LTS via nvm with binaries pinned to ~/.local/bin so they work in every terminal without sourcing anything.
Step 04
Ollama
Installs via official installer, fixes permissions, enables systemd, and starts the API. Handles the WSL2 models-directory ownership bug and adds a ollama-start convenience wrapper.
Step 05
CUDA / GPU
WSL2: pins /usr/lib/wsl/lib first in ldconfig so the CUDA stub is found before any toolkit. Registers Ollama's bundled libs. Native Linux: installs NVIDIA toolkit from the official Ubuntu repo.
Step 06
Model selection
Auto-picks the largest model that fits your VRAM from a live-refreshed catalog. Shows GPU layers, CPU layers, and VRAM usage. Drop into a full numbered picker to choose any model from the list.
Steps 07–12
Stack + agents
oterm terminal chat UI, optional MCP tool bridges on :8010/:8011, four coding agents in a single multi-select menu, shell aliases, PATH setup, and a final validation pass with pass/warn/fail counts.

03  Coding agents + tools

Four agents. Thirteen tools.
Zero cloud required.

Every agent auto-starts Ollama before launching — no manual ollama serve needed. Cloud API keys are optional drop-in overrides.

qc  ·  qwen-code
Qwen Code
Free Qwen OAuth cloud tier (1,000 req/day) with automatic local Ollama fallback. No API key needed. First launch opens browser for qwen.ai sign-in. Toggle between cloud and local with llm-switch.
cx  ·  codex
OpenAI Codex CLI
Runs in --oss --local-provider ollama mode by default. Set OPENAI_API_KEY to route to OpenAI cloud instead. Model auto-pinned to whatever is active in model.conf.
ai  ·  aider
Aider
Git-aware AI pair programmer. Reads your whole codebase, proposes diffs, and applies them. Run from your project directory — it auto-connects to your active Ollama model and warms up the GPU first.
cc  ·  claude-code
Claude Code
Anthropic's CLI agent routed through a LiteLLM proxy to local Ollama on port 4010. Swap in a real ANTHROPIC_API_KEY to switch to actual Claude 3.5 Sonnet. Proxy starts automatically.
Management tools — all installed to ~/.local/bin
llm-switchInteractive model picker — lists installed models, updates model.conf, wires configs for all agents
llm-addBrowse the full model catalog with VRAM requirements. Pull directly from the picker
llm-doctorFull diagnostic — GPU, Ollama API, oterm, MCP bridges, tool scripts, agents. Counts pass/warn/fail
llm-fixAuto-repair: Ollama lib permissions, service restart, Anthropic adapter, MCP bridges, PATH
llm-updateUpgrade Ollama to latest, re-pull active model, update MCP packages in venv
llm-mcp-startInstall mcpo + mcp-server-time on first run, then start bridges on :8010 and :8011
llm-mcp-stopGracefully stop all running mcpo bridge processes using stored PID files
fix-gpuWSL2 GPU troubleshooter — ldconfig, Ollama systemd override, OLLAMA_NUM_GPU=99, test inference
llm-import-modelsRegister local GGUF files with Ollama — writes a Modelfile and calls ollama create
chatStart oterm — ensures Ollama is running and model is warm before opening the UI
llm-setupRe-run the full installer (idempotent — skips already-completed steps)
04  Quick start

Running in under
ten minutes.

Run the installer
Paste the one-liner into a fresh Ubuntu or WSL2 terminal. Needs sudo. Works offline after initial download. Safe to re-run on failure.
bash <(curl -fsSL .../install.sh)
Open a chat
Open a new terminal so aliases load, then type chat. Ollama starts automatically and loads your model onto the GPU.
chat
Start coding
Go to any project folder and launch an agent. It reads your codebase and connects to Ollama — no configuration needed.
cd ~/project && qc
System requirements
  • →Ubuntu 22.04 or 24.04 (native or WSL2)
  • →sudo access
  • →~10 GB free disk (models extra)
  • →Internet connection during install
GPU (optional but fast)
  • →NVIDIA ≥ 6 GB VRAM · CUDA 11+
  • →AMD ≥ 8 GB VRAM · ROCm
  • →WSL2: Windows driver 520+
  • →CPU-only works — just slower
After install
  • →Open new terminal — aliases load
  • →Run llm-doctor to verify
  • →Run llm-setup to re-run anytime
